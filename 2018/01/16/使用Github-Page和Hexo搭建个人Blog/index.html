使用Github Pages和Hexo搭建个人Blog

Posted on 2018-01-16

摘 要

早些时候看见大神们的个人博客时，就萌发了搭建个人博客的想法，这些天开始着手去做，经过各种查找资料，踩这样那样的坑，一个不怎么成熟的个人博客初步完成了。本文介绍了如何使用Github Pages和Hexo搭建一个个人博客。

什么是Github Pages？

GitHub Pages是一个静态站点托管服务。

你可以通过 Automatic Page Generator 在线创建和发布 GitHub Pages。如果你更喜欢本地操作，你可以使用 Mac 或者 Windows 平台的 GitHub App，或者使用 命令行。

Pages 是通过 HTTP 服务的，不是 HTTPS，所以你不应该使用它处理敏感的事务，像发送密码或者信用卡号码。

注意：GitHub Pages 网站是在互联网上公开的，即使它们所在的库是私有的。如果你有敏感的数据在 Page 库，你应该在发布之前删除它。

详细参见 What is GitHub Pages?

为什么使用Hexo？

比较流行的博客框架有很多,比如Jekyll，Hexo，Simple，Octopress等等，这些框架各有好处，选择使用Hexo的理由也非常简单，和Github推荐的Jekyll相比:

  1. Jekyll基于Ruby实现，安装Jekyll需要搭建Ruby环境，在Windows搭建Ruby环境并不是被推荐的，而 Hexo基于NodeJs实现，在Windows上安装NodeJs开发环境很简单。
  2. Hexo的网页渲染速度很快,部署到Github也十分方便。
  3. 当然最主要的原因时Hexo使用非常非常简单，只要按部就班的操作，就能很快完成自己的博客。

Hexo是一个开源的静态博客生成器,用node.js开发,作者是台湾大学生tommy351。了解Hexo

搭建博客

现在准备工作已经结束，让我们开始搭建一个博客吧！

首先，创建一个文件夹作为博客的目录，它就是我们的博客的根目录了,这里我起名为Hexo_Demo。然后进入文件夹，执行hexo初始化命令。

执行后我们会发现文件夹中多了许多东西。

这里我们需要关注的是source文件夹(用于存放博客的源文件)，themes文件夹(用于存放主题)和_config.yml配置文件，这里的_config.yml配置文件是hexo的主配置文件。

让我们来执行一次，看看博客现在的样子吧。

命令 hexo g 是用来将我们写的博客文件(通常是markdown文件)生成静态网页的，这时我们会发现Hexo_Demo文件夹中多public文件夹。


然后执行hexo s命令在本地预览博客，当命令行出现上图log时，说明服务已经启动成功，打开浏览器，在地址栏输入localhost:4000，注意这里是英文的符号。

如果你遇到了我这样的情况

浏览器一直显示loading状态，但无论如何他都无法加载，那么很可能是hexo的默认端口 4000 被占用了，这时请ctrl + c停止服务，使用命令hexo s -p 8080，8080指端口号，这时我们要在地址栏输入localhost:8080。

不出意外，我们就能看到这样的画面了。

这是 hexo 的默认主题。在这里的Hello World文章就是hexo的Quick Start，让我们跟随这里的步骤再写一篇博客吧。

主 题

如果你对hexo的默认主题不满意，那么我们还可以修改主题，当然这也和使用hexo的其他功能一样，非常简单。

你可以浏览这里hexo官网的主题板块，找到你喜欢的主题。这里我们以Ochuunn为例，这是该主题的demo。

打开Ochuunn主题的Github地址，根据下面README的步骤执行就可以了。下面为具体步骤的截图。

  1. 安装主题所需插件

    注意：这一步不是每个主题都必须的，这是该主题需要的插件。

  2. 下载主题

    下载后在themes文件夹中会多了一个文件夹，这个文件夹中是下载的主题的相关文件了。

  3. 修改hexo配置文件
    使用文本编辑器打开Hexo_Demo文件夹中的_config.yml文件。

    修改theme: ochuunn,这里theme的值是themes文件夹下主题文件夹的名称。需要注意的是，yml配置文件的格式要求严格，冒号后要有空格。

完成后，再执行本地预览看看吧。

关于主题的配置文件在themes/主题名文件夹下的_config.yml配置文件中。

上传到Github

安装 hexo-deployer-git。

$ npm install hexo-deployer-git --save

当然这是我的仓库地址，你需要将你的repository的值修改为你的 username.github.io 的仓库地址。
打开仓库，点击Clone or download,将地址复制到配置文件就好了。

然后使用命名:
